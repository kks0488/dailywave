# DailyWave - 창의적 기능 기획서

> "생산성 앱이 아니다. ADHD 뇌를 위한 신경학적 동반자다."

---

## 현재 상태 진단 (솔직하게)

DailyWave는 "또 다른 투두앱"이 **아니다.** 순차적 step 실행 엔진 + 에너지 기반 AI 추천이라는 진짜 고유한 조합이 있다.

**하지만 잠재력의 60%만 실현된 상태:**

| 문제 | 상세 | 영향 |
|------|------|------|
| memoryTracker 미연결 | export 8개, import 0개 | 치명적 - 개인화 불가 |
| AI 함수 3개 미사용 | estimateTaskTime, suggestOptimalTime, analyzeRoutinePatterns | 높음 |
| 루틴 일일 리셋 없음 | done 상태 영구 저장, 날짜 변경 시 리셋 로직 없음 | 치명적 |
| SOP Library 빈 껍데기 | store에 정의, 액션 0개, UI 0개 | 높음 |
| completionHistory 미추적 | analyzeRoutinePatterns 항상 빈 배열 | 높음 |

**먼저 이것들을 연결하는 것이 "5% 기능" 구현의 전제조건이다.**

---

## 시장 컨텍스트

- ADHD 앱 시장: **5억 달러** → 2033년 **18억 달러** (CAGR 15%)
- 2025년 12월 Cell 저널 연구: ADHD 약물은 주의력이 아닌 **보상/각성 시스템**을 활성화
- 기존 경쟁앱들의 치명적 약점: **신경전형적 디자인을 ADHD에 적용하려는 오류**
- 사용자 최대 불만: 수치심 기반 시스템, 텍스트 과다, 구독 피로

## 5% 창의적 기능 제안 (7개)

### Feature 1: "Chaos Dump" - 혼돈 브레인 덤프 AI

**컨셉**: ADHD 뇌는 구조화된 입력을 요구하는 앱에서 마비된다. 완전히 비구조적인 음성/텍스트를 쏟아내면 AI가 자동 분류한다.

**작동 방식**:
```
사용자 입력 (음성 또는 텍스트, 아무 형식이나):
"아 내일까지 보고서 써야하는데 세탁기도 돌려야하고
 점심에 엄마한테 전화하기로 했는데 까먹을 것 같고
 요즘 너무 지치는데 운동을 다시 시작해야 하나..."

AI 자동 분류 결과:
┌─────────────────────────────────────┐
│ 📋 태스크 (파이프라인으로 추가?)     │
│  • 보고서 작성 (기한: 내일)         │
│  • 세탁기 돌리기                    │
│                                     │
│ ⏰ 루틴 (루틴으로 추가?)            │
│  • 점심에 엄마 전화                 │
│                                     │
│ 💭 메모 (저널에 저장?)              │
│  • 피로감, 운동 재개 고민 중        │
└─────────────────────────────────────┘
```

**왜 5%인가**: 대부분의 앱은 이미 구조화된 입력을 요구한다. "제목을 입력하세요", "마감일을 선택하세요"... ADHD 뇌는 그 순간 마비된다. 이것은 **마비 자체를 해결**한다.

**기술적 기반**: Gemini AI + 기존 `parseAICommand()` 확장. memU로 개인 패턴 학습 (예: "엄마 전화"가 반복되면 자동 루틴 제안).

**경쟁 우위**: Tiimo의 AI task breakdown은 이미 구조화된 태스크를 작게 쪼갤 뿐. 이것은 **비구조적 사고 자체**를 받아들인다.

### Feature 2: "Energy Pulse" - 실시간 에너지 적응 UI

**컨셉**: 사용자의 에너지 상태를 추적하고, UI 자체가 에너지에 맞게 변형된다.

**작동 방식**:
```
🔋 Low Energy Mode:
┌──────────────────┐
│                  │
│   지금 할 일:    │
│                  │
│   🧺 세탁기     │
│      돌리기      │
│                  │
│   [시작하기]     │
│                  │
└──────────────────┘
→ 큰 글씨, 1개만 표시, 최소 UI

⚡ High Energy Mode:
┌──────────────────────────────────┐
│ 파이프라인 3개 | 루틴 5개 | AI  │
│ ┌────┐ ┌────┐ ┌────┐           │
│ │보고│ │운동│ │코딩│  ...       │
│ └────┘ └────┘ └────┘           │
│ 다음 스텝: 3장 작성 (25분)     │
│ 최적 시간: 지금! (집중력 피크)  │
└──────────────────────────────────┘
→ 전체 대시보드, 시간 추정, 패턴 제안
```

**에너지 감지 방법**:
1. **수동**: 앱 시작 시 에너지 슬라이더 (현재 이미 있음)
2. **자동 (memU 학습)**: 시간대별 루틴 완료율, 앱 사용 패턴, 스텝 완료 속도로 추론
3. **미래**: Apple Watch 심박 변이도(HRV) 연동

**왜 5%인가**: 모든 생산성 앱의 UI는 정적이다. 에너지가 바닥일 때 복잡한 대시보드를 보면 ADHD 마비가 온다. **UI가 사용자 상태에 반응하는 앱은 없다.**

**기술적 기반**: 기존 에너지 셀렉터 확장 + CSS 변수/테마 전환 + memU 패턴 학습

### Feature 3: "No-Shame Evening" - 후회 없는 저녁 모드

**컨셉**: ADHD인은 하루가 끝나면 "오늘도 아무것도 못 했다"는 수치심에 빠진다. 이 모드는 **완료된 태스크가 아니라 시도한 노력**에 초점을 맞춘다.

**작동 방식**:
```
🌙 저녁 7시 자동 활성화

┌─────────────────────────────────────┐
│  오늘의 DailyWave 🌊               │
│                                     │
│  💪 시도한 것: 8가지               │
│  ✅ 완료한 것: 3가지               │
│  🔄 내일로 넘긴 것: 5가지          │
│     (괜찮아요, 내일이 있으니까)     │
│                                     │
│  📊 이번 주 에너지 패턴:           │
│  월 ████░░ 화 ██████ 수 ███░░░     │
│  → 화요일에 에너지가 가장 높아요   │
│                                     │
│  🧠 AI 인사이트:                   │
│  "오전에 루틴을 잘 지켰어요.       │
│   오후에 에너지가 떨어진 건         │
│   자연스러운 패턴이에요."           │
│                                     │
│  [내일 준비하기] [그냥 쉬기]       │
└─────────────────────────────────────┘
```

**핵심 원칙**:
- 빨간색 경고 없음. "놓친 태스크" 카운터 없음
- "보이지 않는 작업" 인식 (앱 열기, 루틴 체크, 브레인 덤프도 노력)
- 중립적 언어: "실패"가 아닌 "내일로 이동"
- 에너지 패턴 시각화로 자기 이해 도움

**왜 5%인가**: 기존 앱은 미완료 태스크를 빨간색으로 표시하고, streak이 끊기면 경고한다. **수치심 제거에 의도적으로 초점을 맞춘 생산성 앱은 거의 없다.** 2025년 "Gentle Accountability" 트렌드의 핵심.

**기술적 기반**: completionHistory 추적 (현재 미구현) + memU 주간 패턴 분석 + `analyzeRoutinePatterns()` (현재 죽은 코드) 활성화

### Feature 4: "Ghost Crowd" - 유령 바디더블링

**컨셉**: Focusmate 같은 1:1 화상통화 없이, **익명의 다른 사용자들이 지금 작업 중**이라는 사실만으로 동기부여.

**작동 방식**:
```
┌─────────────────────────────────────┐
│  🌊 DailyWave Now                  │
│                                     │
│  ═══════════════════════ 43명 집중중│
│                                     │
│  👻 비슷한 작업:                   │
│  • 12명이 "문서 작성" 중           │
│  • 8명이 "루틴 체크" 중            │
│  • 23명이 "기타 작업" 중           │
│                                     │
│  지금 당신의 작업:                  │
│  📝 보고서 3장 작성                │
│                                     │
│  [함께 시작하기 ▶]                 │
└─────────────────────────────────────┘
```

**핵심 원칙**:
- 비디오 없음, 채팅 없음, 사회적 부담 0
- 완전 익명 (유저네임도 없음)
- 카테고리별 집계만 표시
- "혼자가 아니다"라는 심리적 안정감만 제공

**왜 5%인가**: Focusmate는 사회적 부담이 크다 (카메라 켜기, 목표 공유, 시간 맞추기). ADHD인에게 이것은 또 다른 부담이다. Ghost Crowd는 **사회적 부담 없이 바디더블링의 이점**만 취한다.

**기술적 고려**: WebSocket 기반 실시간 집계. 프라이버시 최우선 (개인 데이터 전송 없음, 카테고리 집계만). 서버 부하 최소화 (heartbeat 기반 카운트).

### Feature 5: "Future Me Letters" - 미래의 나 메시지

**컨셉**: 태스크 시작 시 "미래의 나"에게 짧은 텍스트/음성 메시지를 남긴다. 완료하면 그 메시지가 보상으로 재생된다.

**작동 방식**:
```
[태스크 시작 시]
┌─────────────────────────────────────┐
│  보고서 작성 시작                   │
│                                     │
│  미래의 나에게 한마디:             │
│  ┌───────────────────────────────┐ │
│  │ "이거 끝내면 치킨 먹을 거야  │ │
│  │  화이팅 미래의 나!"           │ │
│  └───────────────────────────────┘ │
│  [🎙 음성] [✏️ 텍스트]  [건너뛰기]│
└─────────────────────────────────────┘

[태스크 완료 시]
┌─────────────────────────────────────┐
│  ✨ 보고서 작성 완료!              │
│                                     │
│  과거의 내가 보낸 메시지:          │
│  💬 "이거 끝내면 치킨 먹을 거야   │
│      화이팅 미래의 나!"            │
│                                     │
│  → 3시간 전의 내가 널 믿었어 🎉   │
│                                     │
│  [다음 태스크] [잠시 쉬기]         │
└─────────────────────────────────────┘
```

**왜 5%인가**: ADHD 연구에서 "temporal self-continuity" (시간적 자아 연속성)는 핵심 주제. ADHD인은 미래의 자신을 **타인처럼 느끼는** 경향이 있다. 이 기능은 **현재의 나와 미래의 나를 연결**하여 동기부여를 만든다. 이런 심리학적 메커니즘을 앱에 구현한 사례는 없다.

**보상 신경과학**: 2025년 Cell 연구에 따르면 ADHD 뇌는 **예측 불가능한 보상**에 더 강하게 반응. 과거의 나 메시지는 완료 시점에 "예상치 못한 보상"이 되어 도파민 경로를 활성화한다.

### Feature 6: "Micro Victory Wall" - 마이크로 성취 갤러리

**컨셉**: ADHD인은 성취를 즉시 잊는다. "오늘 뭐 했지?"라는 질문에 항상 "아무것도"라고 답한다. 모든 작은 승리를 시각적 타임라인으로 보여준다.

**작동 방식**:
```
┌─────────────────────────────────────┐
│  🏆 나의 Victory Wall              │
│                                     │
│  오늘 ════════════════════          │
│  09:15 ☀️ 모닝 루틴 완료           │
│  10:30 📝 보고서 1장 작성          │
│  11:00 🔄 산만함에서 복귀 (3분만에)│
│  12:00 📞 엄마 전화 (잊지 않았다!) │
│  14:30 🧺 세탁기 돌림              │
│                                     │
│  이번 주 ══════════════════         │
│  월: ⭐⭐⭐                         │
│  화: ⭐⭐⭐⭐⭐⭐                    │
│  수: ⭐⭐                           │
│  → 평균 4.3개/일 (지난주 3.1 ↑38%) │
│                                     │
│  🔥 연속 기록: 12일째 앱 열기      │
│  (streak이 끊겨도 아무 벌칙 없음!) │
└─────────────────────────────────────┘
```

**핵심 원칙**:
- 태스크 완료뿐 아니라 **"보이지 않는 노력"** 도 기록
  - 앱 열기 자체가 승리
  - 산만함에서 돌아오기 = 승리
  - 루틴 체크 = 승리
- streak이 끊겨도 **벌칙 없음** (빨간색 없음, 경고 없음)
- 과거 성취를 쉽게 돌아볼 수 있는 타임라인

**왜 5%인가**: 기존 앱의 "성취" 시스템은 streak 기반이다 (Duolingo, Forest). streak이 끊기면 수치심이 온다. Micro Victory Wall은 **모든 행동을 승리로 프레이밍**하되, 끊김에 대한 벌칙이 없다.

**기술적 기반**: completionHistory + memoryTracker (연결 필요) + memU 장기 패턴 저장

### Feature 7: "Flow Recipes" - SOP 워크플로우 마켓

**컨셉**: 커뮤니티가 만든 워크플로우 템플릿을 공유/다운로드. "ADHD를 위한 Notion 템플릿 마켓" 같은 것.

**작동 방식**:
```
┌─────────────────────────────────────┐
│  🍳 Flow Recipes                   │
│                                     │
│  🔥 인기                            │
│  ┌─────────────────────────────┐   │
│  │ ☀️ ADHD 모닝 루틴 (5단계)   │   │
│  │ ⭐ 4.8 (342명 사용중)       │   │
│  │ by @kkaemo                   │   │
│  │ [미리보기] [가져오기]        │   │
│  └─────────────────────────────┘   │
│                                     │
│  📂 카테고리                       │
│  • 아침 루틴 (23)                  │
│  • 업무 집중 (45)                  │
│  • 집안일 (12)                     │
│  • 운동 (18)                       │
│  • 학습 (31)                       │
│                                     │
│  [내 레시피 공유하기]              │
└─────────────────────────────────────┘
```

**왜 5%인가**: Notion 성공의 핵심은 **템플릿 커뮤니티**였다. ADHD 앱에는 이런 공유 생태계가 없다. DailyWave의 Pipeline 데이터 모델은 이미 이것을 지원할 수 있다 (title, steps, color, icon). SOP Library도 이미 데이터 구조가 있다 (비어있을 뿐).

**커뮤니티 효과**:
- 사용자가 콘텐츠를 만듦 → 오가닉 성장
- "나도 이 레시피 쓰고 있다" → 소속감
- 인기 레시피 크리에이터 → 인플루언서 효과
- 바이럴 루프: 레시피 공유 → 새 사용자 유입 → 더 많은 레시피
